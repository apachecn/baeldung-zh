## **4。用前缀`redirect:`重定向**

前面的方法——使用`RedirectView`——由于几个原因不是最理想的。

首先，我们现在耦合到了 Spring API，因为我们在代码中直接使用了`RedirectView`。

第二，我们现在需要从一开始就知道，当实现控制器操作时，结果总是重定向，但情况可能不总是这样。

更好的选择是使用前缀`redirect:`。重定向视图名称像任何其他逻辑视图名称一样被注入到控制器中。**控制器甚至不知道正在发生重定向。**

看起来是这样的:

[PRE3]

当返回带有前缀`redirect:`的视图名称时， `UrlBasedViewResolver`(及其所有子类)会将其识别为需要发生重定向的特殊指示。视图名称的其余部分将用作重定向 URL。

一个快速但重要的注意事项是，当我们在这里使用这个逻辑视图名称— `redirect:/redirectedUrl` —时，我们正在做一个相对于当前 Servlet 上下文的重定向**。**

如果我们需要重定向到一个绝对的 URL，我们可以使用一个名字，比如一个`redirect: http://localhost:8080/spring-redirect-and-forward/redirectedUrl`。

## **8。重定向 HTTP POST 请求**

对于像银行支付这样的用例，我们可能需要重定向 HTTP POST 请求。根据返回的 HTTP 状态代码，POST 请求可以被重定向到 HTTP GET 或 POST。

根据 HTTP 1.1 协议[引用](https://web.archive.org/web/20221004191540/https://tools.ietf.org/html/rfc7238)，状态代码 301(永久移动)和 302(找到)允许将请求方法从 POST 更改为 GET。该规范还定义了相应的 307(临时重定向)和 308(永久重定向)状态代码，不允许将请求方法从 POST 更改为 GET。
